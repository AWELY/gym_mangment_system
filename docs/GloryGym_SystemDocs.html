1. قاعدة البيانات — Database Schema

بناءً على مراجعة الكود الفعلي للنظام، يحتوي النظام على 8 جداول موزّعة كما يلي:

┌── جدول 1: SYSTEM_USERS — مستخدمو النظام ──┐

الحقلالنوعالقيدالوصف
User_ID 🔑INTPRIMARY KEY · AUTO_INCREMENTالمعرّف الفريد
Full_NameNVARCHAR(100)NOT NULLالاسم الكامل للمستخدم
Username 🆔NVARCHAR(50)NOT NULL · UNIQUEاسم الدخول (فريد)
Password_HashNVARCHAR(255)NOT NULLكلمة المرور (مشفّرة)
User_RoleNVARCHAR(20)CHECK IN ('Admin','Receptionist')الصلاحية / الدور
📌 الكود: AppSession.UserRole enum = {Admin, Receptionist} | UserDirectoryEntry

┌── جدول 2: SUBSCRIPTION_PLANS — باقات الاشتراك ──┐

الحقلالنوعالقيدالوصف
Plan_ID 🔑INTPRIMARY KEY · AUTO_INCREMENTالمعرّف الفريد
Plan_Name 🆔NVARCHAR(100)NOT NULL · UNIQUEاسم الباقة
PriceDECIMAL(10,2)NOT NULL · CHECK > 0السعر بالدولار
Duration_ValueINTNOT NULL · CHECK > 0قيمة مدة الاشتراك
Duration_UnitNVARCHAR(10)CHECK IN ('شهر','سنة')وحدة المدة
📌 البيانات الافتراضية: Basic=30$/شهر · Pro=50$/3أشهر · Annual=300$/سنة

┌── جدول 3: TRAINERS — المدربون ──┐

الحقلالنوعالقيدالوصف
Trainer_ID 🔑INTPRIMARY KEY · AUTO_INCREMENTالمعرّف الفريد
Trainer_NameNVARCHAR(100)NOT NULLاسم المدرب
Phone_NumberNVARCHAR(20)NOT NULLرقم الهاتف
SpecialtyNVARCHAR(100)NOT NULLالتخصص الرياضي
Monthly_SalaryDECIMAL(10,2)NOT NULL · CHECK >= 0الراتب الشهري
Join_DateDATENOT NULL · DEFAULT GETDATE()تاريخ الانضمام

┌── جدول 4: MEMBERS — الأعضاء ──┐

الحقلالنوعالقيدالوصف
Member_ID 🔑INTPRIMARY KEY · AUTO_INCREMENTالمعرّف الفريد
Full_NameNVARCHAR(100)NOT NULLالاسم الكامل
Phone_NumberNVARCHAR(20)NULLABLEرقم الهاتف
GenderNVARCHAR(10)CHECK IN ('ذكر','أنثى')الجنس
Join_DateDATENOT NULL · DEFAULT GETDATE()تاريخ الانضمام
Plan_ID 🔗INTFOREIGN KEY → SUBSCRIPTION_PLANSالباقة المشترك بها
Trainer_ID 🔗INTFOREIGN KEY → TRAINERS · NULLABLEالمدرب المُعيَّن
⚠️ تعديل مطلوب: الكود يخزّن PlanName كنص وليس Plan_ID. يجب التحويل لـ FK عند تطبيق DB فعلي.

┌── جدول 5: STORE_PRODUCTS — منتجات المتجر ──┐

الحقلالنوعالقيدالوصف
Product_ID 🔑INTPRIMARY KEY · AUTO_INCREMENTالمعرّف الفريد
Product_Name 🆔NVARCHAR(100)NOT NULL · UNIQUEاسم المنتج
PriceDECIMAL(10,2)NOT NULL · CHECK > 0سعر الوحدة
CategoryNVARCHAR(50)NOT NULLالفئة (بروتين/معدات/...)
Stock_QtyINTNOT NULL · DEFAULT 0 · CHECK >= 0الكمية المتوفرة
Expiry_DateDATENOT NULLتاريخ انتهاء الصلاحية
Photo_Base64TEXTNULLABLEصورة المنتج (Base64)
EmojiNVARCHAR(5)NULLABLEرمز تعبيري للمنتج

┌── جدول 6: STORE_SALES — فواتير البيع ──┐

الحقلالنوعالقيدالوصف
Sale_ID 🔑INTPRIMARY KEY · AUTO_INCREMENTرقم الفاتورة
Sold_AtDATETIMENOT NULL · DEFAULT NOW()وقت وتاريخ البيع
Total_AmountDECIMAL(10,2)NOT NULL · CHECK > 0إجمالي الفاتورة
User_ID 🔗INTFOREIGN KEY → SYSTEM_USERSالمستخدم الذي أجرى البيع
Summary_TextNVARCHAR(500)NULLABLEملخص نصي للمنتجات
⚠️ تعديل مطلوب: الكود لا يسجّل User_ID في الفاتورة حالياً.

┌── جدول 7: SALE_DETAILS — تفاصيل الفاتورة ──┐

الحقلالنوعالقيدالوصف
Detail_ID 🔑INTPRIMARY KEY · AUTO_INCREMENTالمعرّف الفريد
Sale_ID 🔗INTFOREIGN KEY → STORE_SALESرقم الفاتورة
Product_ID 🔗INTFOREIGN KEY → STORE_PRODUCTSرقم المنتج المباع
QuantityINTNOT NULL · CHECK > 0الكمية المباعة
Unit_PriceDECIMAL(10,2)NOT NULLسعر الوحدة وقت البيع
⚠️ تعديل مطلوب: الكود يخزّن ملخصاً نصياً فقط. يجب استبداله بهذا الجدول.

┌── جدول 8: FEEDING_PLANS — خطط التغذية (إضافة جديدة) ──┐

الحقلالنوعالقيدالوصف
FPlan_ID 🔑INTPRIMARY KEY · AUTO_INCREMENTالمعرّف الفريد
Plan_Name 🆔NVARCHAR(100)NOT NULL · UNIQUEاسم خطة التغذية
PDF_PathNVARCHAR(500)NOT NULLمسار ملف PDF

7. التعديلات المقترحة على قاعدة البيانات

#الوضع الحاليالتعديل المطلوبالأولوية
1 MemberRecord.PlanName مخزّن كـ string نصي تحويل إلى Plan_ID (FK → SUBSCRIPTION_PLANS) 🔴 عالية
2 لا يوجد Trainer_ID مرتبط بالعضو في الكود إضافة حقل Trainer_ID (FK NULLABLE) لجدول Members 🟡 متوسطة
3 StoreSaleRecord لا يسجّل من أجرى البيع استخدام AppSession.Username وربطه بـ User_ID FK 🔴 عالية
4 تفاصيل الفاتورة كملخص نصي فقط (Summary) استبدالها بجدول SALE_DETAILS للتتبع الدقيق 🔴 عالية
5 FeedingPlanRecord موجود لكن غير في Schema إضافة جدول FEEDING_PLANS رسمياً 🟡 متوسطة
6 كلمات المرور مخزّنة كـ plain text تشفير بـ SHA-256 أو BCrypt قبل التخزين 🔴 أمان حرج
7 جميع البيانات في الذاكرة (in-memory) تُفقد عند الإغلاق ربط بقاعدة بيانات حقيقية (SQLite أو SQL Server) 🔴 عالية جداً
